# Five-Letter Word Frequency Site Design

**GitHub Issue:** None

## Summary

This project builds a static informational website that displays letter frequency statistics derived from the Unix system word list (`/usr/share/dict/words`), filtered to roughly 4,700 lowercase five-letter words. A Python script (`main.py`) uses Polars to compute four frequency views — overall letter counts, positional unigrams, adjacent-letter bigrams, and gap-letter trigrams — and generates a Markdown page with embedded HTML tables and heatmap grids. The site is built by Zensical and deployed to GitHub Pages via GitHub Actions.

The design favours static HTML over client-side rendering. Overall frequencies, unigrams, and bigrams are pure static HTML — pre-rendered coloured tables with `<details>` elements for drill-downs. The only JavaScript is a small script for trigram cell click-to-expand, where pre-rendering all ~3,000 non-empty completion lists would make the page too heavy. All computation happens at build time in Python.

## Definition of Done

1. Static site builds and deploys to GitHub Pages via GitHub Actions.
2. Page displays overall letter frequency, positional unigram, bigram, and trigram frequency data computed from `/usr/share/dict/words` (lowercase 5-letter words only).
3. Bigram row headers are clickable (via `<details>`), expanding to show ±1 neighbour frequency distributions.
4. Trigram heatmap cells are clickable (via JS), expanding to show the full completion list with counts.
5. Page loads with no external runtime dependencies (no CDN). Only JS is a small trigram-expand script.

## Glossary

- **Polars**: A Python data manipulation library designed for high performance on tabular data. Used here to compute all frequency matrices from the word list.
- **Zensical**: The static site generator driving this project. Takes Markdown source files under `docs/`, applies configuration from `zensical.toml`, and writes the built site to `site/`. Successor to mkdocs-material by squidfunk.
- **uv**: A Python package and project manager. Used to install dependencies (`uv sync`) and run project scripts (`uv run`).
- **unigram (positional)**: The frequency of a single letter at each of the five word positions. Rendered as a 26x5 heatmap.
- **bigram**: A pair of adjacent letters at specific word positions. 4 grids (positions 1-2, 2-3, 3-4, 4-5), each 26x26. Rows = letter at the earlier position, columns = letter at the later position.
- **trigram (gap)**: A three-letter pattern where two positions are known and one (the "gap") is the variable. 9 combinations: 3 gap positions x 3 word windows.
- **heatmap**: A grid where each cell's background colour encodes its numeric value. Cells coloured by frequency relative to the grid's maximum.
- **drill-down**: A `<details>` expansion triggered by clicking a bigram row header, showing the frequency distribution of neighbouring positions for that letter.
- **data contract**: The agreed JSON structure that `main.py` produces and the page consumes.

## Architecture

Single-page static site generated by Zensical. Frequency data is pre-computed by `main.py` (Polars) and embedded into the page as static HTML. Only the trigram click-to-expand uses JavaScript.

**Data pipeline:** `main.py` reads `/usr/share/dict/words`, filters to lowercase 5-letter words (~4,700 words), computes all frequencies with Polars, and generates `docs/index.md` containing static HTML tables with inline styles for heatmap colouring. Trigram data is written to `docs/data/trigrams.json` for JS consumption.

**Frontend:** `docs/index.md` is the generated page. It contains:
- Static HTML tables for overall, unigram, and bigram sections
- `<details>` elements for bigram collapsible grids and row drill-downs
- Coloured `<td>` cells with inline `background-color` for heatmap effect
- Placeholder `<div>`s in the trigram section, populated by JS on click

**JavaScript:** `docs/js/trigram-expand.js` — a small script (~50 lines) that loads `docs/data/trigrams.json` on first click and populates completion lists. This is the only JS on the page.

**CSS:** `docs/css/heatmap.css` — styles for heatmap tables, frequency bars, drill-down expansions, and trigram completion lists.

**Build flow:** GitHub Actions runs `uv sync` → `uv run python main.py` → `uv run zensical build --clean`, deploying `site/` to GitHub Pages.

### Data contracts

**Trigram JSON** (the only runtime data file):

**`docs/data/trigrams.json`**
```json
{
  "gap1_win123": {"b": {"c": {"a": 5, "d": 2, ...}, ...}, ...},
  "gap2_win123": {...},
  "gap3_win123": {...},
  "gap1_win234": {...},
  "gap2_win234": {...},
  "gap3_win234": {...},
  "gap1_win345": {...},
  "gap2_win345": {...},
  "gap3_win345": {...}
}
```
9 keys (3 gap positions x 3 word windows). Each is a nested dict: known_letter_1 → known_letter_2 → {gap_letter: count, ...}. Zero-count entries are omitted.

All other data (overall, unigrams, bigrams) is baked into the HTML at build time — no JSON files needed at runtime.

## Additional Considerations

**Empty cells:** Many bigram and trigram combinations will have zero occurrences. These render as blank/grey cells, keeping the visual clean while showing sparsity patterns.

**Data volume:** Only `trigrams.json` is loaded at runtime. With ~4,700 words and sparse trigram space (~3,000 non-empty cells across 9 combos), this file is expected to be moderate in size. All other data is baked into the HTML.

**Deployment early:** Phase 1 validates the full pipeline (generate → build → deploy) with minimal content, addressing the risk of discovering Zensical deployment issues late.
